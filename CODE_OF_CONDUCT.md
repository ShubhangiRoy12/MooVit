# MooVit Code of Conduct

## Our Commitment

MooVit is committed to providing a welcoming, safe, and respectful environment for everyone who participates in the project. This applies regardless of age, body size, disability, ethnicity, gender identity or expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

We want MooVit to be a community where contributors can collaborate openly, ask questions without fear, and improve the project together.

## Expected Behavior

Examples of behavior that help create a positive environment include:

- Showing empathy and respect toward other people
- Giving and receiving constructive feedback professionally
- Being patient with newcomers and contributors from different backgrounds
- Focusing discussions on the work, not on the person
- Respecting different viewpoints and experiences
- Taking responsibility for mistakes and learning from them

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, bullying, intimidation, or discrimination in any form
- Insults, personal attacks, or derogatory comments
- Sexualized language, imagery, or unwelcome advances
- Trolling, deliberate provocation, or sustained disruption of discussions
- Publishing private information without explicit permission
- Spam, unrelated self-promotion, or repeated off-topic content
- Plagiarism, impersonation, or other dishonest conduct

## Scope

This Code of Conduct applies in all MooVit community spaces, including:

- GitHub issues, pull requests, commits, comments, and discussions
- Project documentation, code review, and contributor conversations
- Official project communication channels and community events
- Public spaces when someone is representing MooVit or its community

## Reporting

If you experience or witness behavior that violates this Code of Conduct, report it through one of the following channels:

- Open a private report with the maintainers through the repository's contact or discussion channels
- If a private channel is not available, open a GitHub issue requesting maintainer contact without sharing sensitive details publicly

When reporting, include relevant context such as what happened, where it happened, and any supporting screenshots or links if safe to provide.

All good-faith reports will be reviewed as promptly and confidentially as possible.

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may remove, edit, or reject comments, commits, code, issues, and other contributions that do not align with these standards.

Depending on severity and pattern of behavior, maintainers may take actions including:

1. A private warning
2. A request to stop the behavior
3. Temporary restriction from project participation
4. Permanent removal from project spaces

## Maintainer Responsibilities

Maintainers should:

- Lead by example in project interactions
- Apply the rules consistently and fairly
- Prioritize the safety of affected contributors
- Avoid public arguments about active reports when privacy is needed
- Document decisions internally when enforcement action is taken
